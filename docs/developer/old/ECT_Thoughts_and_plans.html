ECT: Thoughts and Plans

Chris Greenhalgh, 2004-09-13 [updated 2004-09-21]

Status

  • Java WebStart-able system
  • Distributed (single LAN Installation)
  • Device proxies for: EZIO, TagIt, ACCORD SmartIts, Motes (partial), X10 (in progress), simple Joystick (in progress), Phidgets (output only)
  • Audio and video capture and simple processing framework.
  • Dynamo display surface.
  • Hosting/import for Processing applets
  • Computation/behaviour through: simple script, association learner, delay-line, counter-timer.
  • Persistence and re-starting
  • Graph-view editor, tree-view editor, configuration save/restore
  • Recording of dataspace, supporting potential replay/review

Limitations of current implementation

  • Authoring only using explicit component/line GUIs
    • Slow and unwieldy for larger systems
    • Visual complexity increases rapidly
  • No self-discovery/plug-and-play
    • Explicitly create all components
    • Explicitly identify COM ports for serial devices
  • Graph-view editor does not cope well with restart/joining a running application
  • No composite components or views
  • No fail-over, build-in adaptation, etc.
  • No examples of or particular support for adaptation by components (e.g. component creation, selection, re/wiring)
  • No recording of non-dataspace state/events (for replay/review)
  • No support for mobile/ad-hoc style join/leave or federation
  • Partially centralised dataspace implementation
  • No remote diagnostic support
  • No support for auto-instantiation
  • No support for singletons or other re-use of components across multiple requests
  • No additional information about properties for framework or editors, e.g. input/output, reliable/unreliable, update rate
  • No fan-in policy/framework
  • No service UI story
  • No array type property support
  • Effectively Java-only, with no standard network protocol (undocumented and not self-describing/extensible, anyway)
  • Graph editor does not reflect addition/removal of dynamic properties for components on the surface

Limitations of use/experience

  • No "real" use
  • No specific success metrics
  • No experimental data/results/analysis
